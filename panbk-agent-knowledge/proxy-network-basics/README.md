## 0. 先用一句话建立整体认知

科学上网不是一个单独软件，也不是一个神秘按钮，而是一套网络转发方案：

```text
你的电脑/手机
  -> 客户端
  -> 节点
  -> 目标网站
```

更完整一点：

```text
机场提供订阅
订阅里面有很多节点
客户端导入订阅并连接节点
你的流量通过节点转发到目标网站
目标网站看到的是节点的出口 IP
```

## 1. 术语速查表

| 名词 | 小白解释 | 更准确的理解 | 你日常会怎么遇到 |
|---|---|---|---|
| 梯子 | 科学上网工具的俗称 | 一套代理访问方案 | “我的梯子能不能上 ChatGPT？” |
| 代理 | 中转服务器机制 | 请求先到代理，再由代理访问目标 | 浏览器/终端走 `127.0.0.1:7890` |
| 客户端 | 本机连接工具 | 管理节点、规则、端口、Tun、DNS | Clash、v2rayN、Shadowrocket |
| 机场 | 节点服务商 | 提供订阅、节点、套餐、面板 | 买套餐、看剩余流量 |
| 订阅 | 节点配置链接 | 一个 URL，里面包含节点列表 | 导入客户端、更新订阅 |
| 节点 | 代理服务器 | 真正帮你出国访问网站的服务器 | 香港节点、日本节点、美国节点 |
| VPS | 云服务器 | 可用来自建节点的基础设施 | Vultr、AWS、Oracle Cloud |
| 协议 | 客户端和节点的通信方式 | SS、VLESS、Trojan、Hysteria2 等 | 订阅里自动带着 |
| 分流 | 哪些流量走代理，哪些直连 | 规则模式/全局模式/直连模式 | 国内网站直连，国外网站代理 |
| Tun | 接管系统网络的模式 | 让不支持系统代理的软件也能走代理 | 游戏、终端、Docker 常用 |
| 链式代理 | 代理后面再接代理 | 多跳路径 | `你 -> 节点 A -> 节点 B -> 网站` |

## 2. 什么是代理

代理就是“中间人服务器”。

没有代理时：

```text
你 -> 网站
```

使用代理后：

```text
你 -> 代理服务器 -> 网站
```

这会产生几个重要效果：

- 目标网站看到的是代理服务器的 IP，不是你本机网络的出口 IP。
- 你的设备只需要能连上代理服务器，后面的访问由代理服务器完成。
- 代理服务器在哪里，你通常就表现为哪个地区的访问者。

例如你连了日本节点：

```text
你在中国大陆
  -> 日本节点
  -> YouTube / Google / ChatGPT
```

目标网站一般会看到“日本 IP 正在访问”。

### 代理不等于绝对匿名

代理能改变出口路径，但不能保证绝对匿名。以下信息仍可能暴露或被关联：

- 登录账号本身
- 浏览器指纹
- Cookie
- DNS 或 WebRTC 泄露
- 机场服务商日志策略
- 目标网站对节点 IP 的风控记录

所以代理更准确地说是“改变网络出口和转发路径”，不是“万能隐身”。

---

## 3. 什么是梯子

“梯子”是中文网络语，通常指能帮助访问国际互联网服务的一整套工具和配置。

它通常由这些部分组成：

```text
梯子
= 客户端
+ 订阅
+ 节点
+ 协议
+ 分流规则
+ DNS 配置
```

所以当别人说“我有一个梯子”，可能指的是：

- 他买了一个机场套餐；
- 他拿到了一个订阅链接；
- 他把订阅导入了 Clash、v2rayN、Shadowrocket 等客户端；
- 他选择某个节点后，可以访问 Google、YouTube、ChatGPT 等服务。

严格来说，“梯子”不是一个具体技术名词，而是对整套代理使用环境的俗称。

---

## 4. 什么是客户端

客户端是你设备上真正操作梯子的地方。

常见客户端：

| 平台 | 常见客户端 |
|---|---|
| Windows | Clash Verge Rev、v2rayN、NekoRay |
| macOS | Clash Verge Rev、ClashX Meta、Surge |
| iOS | Shadowrocket、Quantumult X、Stash、Surge |
| Android | v2rayNG、Clash Meta for Android、NekoBox |
| Linux | Mihomo、Clash Verge Rev、NekoRay |

客户端负责：

- 导入机场订阅；
- 展示节点列表；
- 选择节点；
- 启动本地代理端口；
- 控制系统代理；
- 提供规则模式、全局模式、直连模式；
- 处理 DNS；
- 开启 Tun 模式；
- 统计连接、延迟和流量。

### 本地代理端口是什么意思

很多客户端会在你电脑上开一个本地端口，例如：

```text
127.0.0.1:7890
127.0.0.1:10808
```

`127.0.0.1` 代表“你自己的电脑”。端口代表客户端监听的位置。

当浏览器或终端配置为使用这个端口时，请求会先交给客户端：

```text
浏览器 / 终端
  -> 127.0.0.1:7890
  -> 客户端
  -> 远程节点
  -> 目标网站
```

结合你之前的使用习惯，你的终端环境里曾经出现过 `10808` 和 `7890` 两类本地端口。它们不是“两个机场”，而是本机不同代理入口。真正通往外网的是客户端后面选中的远程节点。

---

## 5. 什么是机场

机场是提供代理节点的服务商。

你可以把机场理解成：

```text
机场 = 节点供应商 + 用户面板 + 流量套餐 + 订阅系统
```

机场通常提供：

- 多个地区节点；
- 一个订阅链接；
- 每月流量额度；
- 节点倍率；
- 在线面板；
- 流媒体解锁节点；
- ChatGPT / TikTok / Netflix 等特定用途节点；
- 工单、公告和续费入口。

机场不是客户端。机场也不是一个具体节点。

更准确的关系是：

```text
机场给你订阅链接
订阅链接里包含节点配置
客户端导入订阅
你在客户端里选择节点
```

---

## 6. 什么是订阅

订阅通常是一个 URL。

形式可能类似：

```text
https://example.com/api/v1/client/subscribe?token=xxxxxxxx
```

它里面包含你的节点配置。你不需要手动复制每个节点的服务器地址、端口、密码、协议，只要把订阅导入客户端即可。

### 订阅链接很敏感

订阅链接相当于你机场账号的“钥匙”之一。

泄露后别人可能：

- 导入你的节点；
- 消耗你的流量；
- 导致你的账号异常；
- 让机场误判为多人共享。

如果订阅泄露，应该去机场面板重置订阅链接。

---

## 7. 什么是节点

节点就是代理服务器。

日常看到的“香港 01”“日本东京”“美国洛杉矶”“新加坡 IEPL”都属于节点。

一个节点一般包含：

- 服务器地址；
- 端口；
- 协议；
- 密码或 UUID；
- 加密方式；
- 传输层配置；
- SNI / Host / 伪装域名；
- 是否支持 UDP；
- 倍率；
- 地区标签。

小白不需要手动理解每个字段。正常情况下，机场订阅会自动把这些配置交给客户端。

### 怎么选节点

常见选择逻辑：

| 场景 | 推荐节点 |
|---|---|
| 日常网页、搜索、ChatGPT | 香港、日本、新加坡、台湾、美国节点都可测试 |
| YouTube、Netflix | 优先速度稳定、支持流媒体解锁的节点 |
| 游戏 | 优先低延迟、低丢包节点 |
| 下载 | 优先低倍率、高带宽节点 |
| 开发下载 GitHub/npm | 优先稳定节点，不一定最低延迟 |
| TikTok | 常需要对应地区节点和移动端环境配合 |

不要只看延迟。延迟低不代表一定快，也不代表一定能解锁目标服务。

---

## 8. 什么是协议

协议是客户端和节点之间“说话的规则”。

常见协议包括：

- Shadowsocks，简称 SS；
- VMess；
- VLESS；
- Trojan；
- Hysteria2；
- TUIC；
- WireGuard；
- SOCKS5；
- HTTP Proxy。

类比：

```text
客户端和节点之间要通信
协议就是它们共同使用的语言
```

你通常不需要手动选择协议。机场订阅里已经配好了，只要客户端支持对应协议即可。

---

## 9. 什么是 VPS

VPS 是 Virtual Private Server，虚拟专用服务器。

它是一台你租来的云服务器。常见云服务商包括：

- Vultr；
- DigitalOcean；
- Linode；
- AWS；
- Google Cloud；
- Oracle Cloud；
- Hetzner；
- 搬瓦工等。

VPS 本身不是梯子，但你可以在 VPS 上部署代理服务，让它变成一个节点。

关系可以这样理解：

```text
VPS = 原材料
节点 = 在 VPS 或服务器上搭好的代理服务
机场 = 批量提供节点的服务商
客户端 = 你本机连接节点的软件
```

### 机场和自建 VPS 的区别

| 方案 | 优点 | 缺点 |
|---|---|---|
| 买机场 | 简单、节点多、维护少 | 依赖服务商、共享环境、隐私取决于机场 |
| 自建 VPS | 自己控制、独享资源、逻辑清楚 | 要懂服务器、地区少、IP 被封要自己处理 |

对纯小白来说，建议先理解机场和客户端，再考虑 VPS 自建。

## 10. 什么是链式代理

普通代理：

```text
你 -> 节点 A -> 网站
```

链式代理：

```text
你 -> 节点 A -> 节点 B -> 网站
```

路径对比图：

![直连、普通代理和链式代理路径对比](assets/02-traffic-paths.svg)

链式代理的常见用途：

- 让一个代理作为入口，另一个代理作为出口；
- 公司/校园代理后面再接机场；
- 本地工具只能连 SOCKS5，于是先连本地代理，再转到远程节点；
- 特殊网络环境下绕过某个限制；
- 需要固定最终出口地区。

缺点：

- 更慢；
- 延迟更高；
- 掉线概率更高；
- DNS、UDP、证书、规则问题更难查；
- 小白容易不知道请求到底从哪里出去。

建议：没有明确需求时，不要优先使用链式代理。先把普通代理跑稳定。

## 11. 系统代理、规则模式、全局模式、直连模式、Tun

### 系统代理

系统代理是操作系统提供给应用的代理设置。

开启后，大多数遵守系统代理的软件会走客户端。

但有些软件不听系统代理，例如：

- 某些游戏；
- 某些命令行程序；
- Docker；
- 部分开发工具；
- 某些 Electron 应用。

### 规则模式

规则模式是最适合日常使用的模式。

它大概这样工作：

```text
国内网站 -> 直连
国外网站 -> 代理
广告域名 -> 拦截
局域网地址 -> 直连
```

优点：

- 国内网站速度快；
- 国外网站可访问；
- 不容易让所有流量都绕远路；
- 更符合日常使用。

### 全局模式

全局模式会让大部分流量都走代理。

适合排查问题：

```text
规则模式打不开
切全局后能打开
说明很可能是分流规则或 DNS 问题
```

不建议长期无脑全局，因为国内服务可能变慢，甚至触发异常登录风控。

### 直连模式

直连模式就是全部不走代理。

适合测试本地网络是否正常。

### Tun 模式

Tun 模式比系统代理更强，它会在网络层接管流量。

适合：

- 游戏；
- 终端；
- Docker；
- 不支持系统代理的软件；
- 需要更完整接管网络的场景。

风险：

- 可能影响局域网访问；
- 可能和公司 VPN 冲突；
- 可能导致 DNS 行为变化；
- 配错时会让网络表现更混乱。

建议：浏览器能用时，先别急着开 Tun。只有当“某个软件不走代理”时，再考虑 Tun。

## 12. DNS 为什么重要

DNS 负责把域名翻译成 IP。

例如：

```text
google.com -> 某个 IP 地址
```

科学上网里 DNS 很重要，因为：

- DNS 可能被污染；
- 国内外解析结果可能不同；
- 分流规则需要根据域名判断走代理还是直连；
- 某些网站依赖正确地区的 DNS 结果；
- 错误 DNS 会导致“节点通，但网站不通”。

常见现象：

- Google 打不开，但 Telegram 能用；
- 某些网站跳到奇怪页面；
- ChatGPT 页面能开，但接口报错；
- 节点测速正常，但浏览器打不开网页。

建议：

- 小白优先使用客户端默认 DNS 配置；
- 不要同时在系统、浏览器、客户端里乱改 DNS；
- 排查时可以用“规则模式 vs 全局模式”判断是否是规则/DNS 问题。

## 13. 流量、倍率、延迟、速度、丢包

### 流量

机场套餐通常按流量计费。

例如每月 100GB，意思是你通过节点转发的数据总量有额度。

### 倍率

倍率决定流量扣除速度。

```text
1x：使用 1GB，扣 1GB
2x：使用 1GB，扣 2GB
0.5x：使用 1GB，扣 0.5GB
```

高倍率节点不一定更好，但经常代表线路更贵或更稀缺。

### 延迟

延迟是你到节点的响应时间，单位通常是 ms。

一般规律：

- 香港、日本、新加坡对中国大陆延迟较低；
- 美国、欧洲延迟较高；
- 游戏和实时通话更看重延迟。

### 速度

速度决定看视频、下载文件是否流畅。

速度和延迟不是一回事。一个节点延迟低，但带宽可能很差。

### 丢包

丢包是数据传输过程中丢失。

丢包高时会出现：

- 视频卡顿；
- 网页加载到一半；
- 游戏延迟突然飘；
- SSH 或下载中断。

## 14. 针对你的使用场景：已有梯子环境怎么理解和使用

结合你已有的环境，重点关注三层：

```text
第一层：客户端有没有正常运行
第二层：本地代理端口是否对应正确
第三层：远程节点是否适合当前目标
```

你之前的知识文档里已经记录过终端代理端口切换习惯：

```bash
proxy_status
proxy_on
proxy_7890
proxy_off
```

这说明你的使用场景不只是浏览器，还包括终端下载、npm、GitHub、curl 等开发场景。

### 浏览器使用建议

推荐：

- 客户端使用规则模式；
- 节点先选香港、日本、新加坡；
- ChatGPT 不通时换美国、日本、新加坡节点；
- YouTube 卡顿时换速度更高或低倍率节点；
- 不要频繁改 DNS。

### 终端使用建议

终端不会天然知道你的客户端在哪里。它通常靠环境变量走代理：

```bash
HTTP_PROXY=http://127.0.0.1:7890
HTTPS_PROXY=http://127.0.0.1:7890
ALL_PROXY=socks5://127.0.0.1:7890
```

如果你的客户端实际监听的是 `10808`，但终端指向 `7890`，就会失败。

所以终端排查先看：

```bash
proxy_status
```

然后确认：

- 当前环境变量指向哪个端口；
- 客户端是否真的开了这个端口；
- 当前节点是否可用；
- 目标站是否需要特定地区节点。

### 开发下载场景建议

例如 GitHub、npm、bun、pip、Docker 拉取失败时，先不要急着换软件源。

按顺序查：

1. 浏览器能不能打开目标网站；
2. 终端代理变量是否正确；
3. 当前端口是否可用；
4. 当前节点是否可用；
5. 换 `7890` 或 `10808`；
6. 换节点；
7. 再考虑软件源或证书问题。

## 15. 新手排查流程

打不开网站时，不要同时乱改节点、DNS、规则、Tun。按顺序排查：

![科学上网问题排查流程图](assets/03-troubleshooting-flow.svg)

文字版：

1. 客户端是否运行？
2. 系统代理是否开启？
3. 本地代理端口是否正确？
4. 当前节点是否可用？
5. 是否只有某个网站打不开？
6. 切全局模式是否能打开？
7. 换节点是否能打开？
8. 目标软件是否不听系统代理？
9. 是否需要 Tun 或终端环境变量？

### 快速测试命令

测试某个本地 HTTP 代理端口：

```bash
curl -I --proxy http://127.0.0.1:7890 https://www.google.com
```

测试另一个端口：

```bash
curl -I --proxy http://127.0.0.1:10808 https://www.google.com
```

如果 `7890` 通而 `10808` 不通，说明问题可能在端口配置或客户端监听上。  
如果两个都不通，先确认客户端是否运行、节点是否连通。  
如果 Google 通但某个网站不通，可能是目标网站屏蔽节点、DNS、规则或账号风控。

## 16. 常见误区

### 误区 1：机场就是客户端

不是。机场是服务商，客户端是你本机软件。

### 误区 2：节点延迟最低就最好

不一定。看视频更看速度，游戏更看低延迟和低丢包，ChatGPT 更看出口 IP 是否干净。

### 误区 3：全局模式一定更好

不一定。全局适合排查，日常更推荐规则模式。

### 误区 4：开了客户端，所有软件都会走代理

不一定。有的软件不听系统代理，需要 Tun 或单独配置代理。

### 误区 5：VPS 一定比机场好

不一定。VPS 更可控，但维护成本高，节点少，IP 被封需要自己处理。

### 误区 6：链式代理更高级，所以应该一直开

不建议。链式代理增加复杂度，除非你知道自己为什么需要它。

```text

  -> 节点 A
```

## 18. 最小行动建议

对你当前阶段，建议这样用：

1. 日常浏览：客户端开规则模式，选择香港/日本/新加坡节点。
2. ChatGPT 不通：优先换美国、日本、新加坡、台湾节点。
3. YouTube 卡：换低倍率高速度节点，不只看延迟。
4. 终端下载失败：先执行 `proxy_status`，确认端口和客户端一致。
5. 某个软件不走代理：再考虑 Tun，不要一开始就开很多高级功能。
6. 想理解更深：再学习 VPS 自建、DNS 策略、链式代理、规则集。
